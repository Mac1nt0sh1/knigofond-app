# Переход на SQLite

## Преимущества SQLite:

- ✅ **Быстро** - нет задержек на подключение
- ✅ **Просто** - один файл, не нужны внешние сервисы
- ✅ **Локально** - все данные на вашем компьютере
- ✅ **Бесплатно** - не нужно настраивать облачные сервисы
- ✅ **Идеально для разработки** - быстрое тестирование

## Шаг 1: Обновите .env файл

Откройте `.env` и измените `DATABASE_URL`:

```env
# SQLite - локальная база данных
DATABASE_URL="file:./dev.db"
```

**Важно:** Удалите или закомментируйте `DIRECT_URL` - он не нужен для SQLite.

## Шаг 2: Перегенерируйте Prisma Client

```bash
npx prisma generate
```

## Шаг 3: Создайте базу данных

```bash
npm run db:push
```

Это создаст файл `prisma/dev.db` с всеми таблицами.

## Шаг 4: Проверьте подключение

```bash
npm run db:check
```

## Что изменилось:

1. **schema.prisma** - изменен provider на `sqlite`
2. **DATABASE_URL** - теперь указывает на файл `file:./dev.db`
3. **DIRECT_URL** - больше не нужен

## Файл базы данных:

- **Расположение**: `prisma/dev.db` (или путь из DATABASE_URL)
- **Размер**: будет расти по мере добавления данных
- **Резервное копирование**: просто скопируйте файл `dev.db`

## Для продакшена:

SQLite подходит для:
- ✅ Локальной разработки
- ✅ Небольших проектов
- ✅ Прототипирования

Для продакшена рекомендуется:
- PostgreSQL (NeonDB, Supabase)
- MySQL
- Другие production-ready базы данных

## Полезные команды:

```bash
# Просмотр базы данных
npm run db:studio

# Выполнение SQL запросов
npm run db:sql "SELECT * FROM User"

# Проверка статуса
npm run db:check
```
